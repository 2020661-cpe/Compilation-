36   <a name="l16"><span class="ln">16   </span></a>The ``distro`` package (``distro`` stands for Linux Distribution) provides 
37   <a name="l17"><span class="ln">17   </span></a>information about the Linux distribution it runs on, such as a reliable 
38   <a name="l18"><span class="ln">18   </span></a>machine-readable distro ID, or version information. 
39   <a name="l19"><span class="ln">19   </span></a> 
40   <a name="l20"><span class="ln">20   </span></a>It is the recommended replacement for Python's original 
41   <a name="l21"><span class="ln">21   </span></a>:py:func:`platform.linux_distribution` function, but it provides much more 
42   <a name="l22"><span class="ln">22   </span></a>functionality. An alternative implementation became necessary because Python 
43   <a name="l23"><span class="ln">23   </span></a>3.5 deprecated this function, and Python 3.8 removed it altogether. Its 
44   <a name="l24"><span class="ln">24   </span></a>predecessor function :py:func:`platform.dist` was already deprecated since 
45   <a name="l25"><span class="ln">25   </span></a>Python 2.6 and removed in Python 3.8. Still, there are many cases in which 
46   <a name="l26"><span class="ln">26   </span></a>access to OS distribution information is needed. See `Python issue 1322 
47   <a name="l27"><span class="ln">27   </span></a>&lt;https://bugs.python.org/issue1322&gt;`_ for more information.
